                  # 推荐系统实践

                  2019-07-09 15:18:16

                  # 概述

                  • 分类

                    • 按数据✅：协同过滤、内容过滤、社会化过滤
                    • 按算法：邻域、图、矩阵分解/概率模型

                  • 推荐系统：联系用户和信息，在用户没有明确需求时，通过分析用户的历史行为给用户的兴趣建模，主动给用户推荐能够满足他们兴趣和需求的信息。

                    • 可以更好地发掘物品的长尾（long tail， 不热门的商品）
                    • 应用：电子商务、视频电影、个性音乐电台、社交网站、个性化阅读、基于位置的服务（附近餐馆etc）、个性化邮件、个性化广告
                    • 评测：
                      • 好的推荐系统：1️⃣准确预测用户的行为，2️⃣扩展用户的视野，帮助用户发现那些他们可能会感兴趣，但却不那么容易发现的东西。3️⃣发掘长尾。
                      • 实验方法：离线实验、用户调查、在线实验（AB test）。
                      • 预测准确度：
                        • 评分预测： 均方根误差（RMSE）、平均绝对误差（MAE）
                        • TopN推荐✅：准确率（precision）、召回率（recall）
                      • 覆盖率：研究物品在推荐列表中出现次数的分布描述推荐系统挖掘长尾的能力，信息熵 or 基尼系数
                      • 多样性：描述推荐列表中物品两两之间的不相似性
                      • 新颖性：最简单的方法是利用推荐结果的平均流行度
                      • 离线实验的优化目标是: 最大化预测准确度 使得 覆盖率 > A, 多样性 > B, 新颖性 > C

                  # 仅利用用户行为数据（协同过滤算法）

                  • 协同过滤算法
                    • 基于邻域的方法（neighborhood-based）
                      • 基于用户：给用户推荐和他兴趣相似的其他用户喜欢的物品
                        • 1.计算用户间兴趣相似度w_uv：Jaccard、余弦；
                          • 改进：物品-用户倒排表
                        • 2.根据（UserCF算法中用户u对物品i的感兴趣程度 p(u,i) = sum(w_uv * 用户v对物品i的兴趣 r_vi) for v ∈ 兴趣最相似的K个用户）推荐物品。
                        • 改进：两个用户对冷门物品采取过同样的行为更能说明他们兴趣的相似度，User-IIF算法惩罚热门物品。
                        • 相比ItemCF使用不多，Digg给用户推荐之前顶过相同文章的用户最近顶的文章；推荐反映小群体兴趣，更社会化；新闻推荐。
                      • 基于物品：给用户推荐和他之前喜欢的物品相似（喜欢A的用户大都也喜欢B）的物品
                        • 1.计算物品之间的相似度w_ij：余弦；
                        • 2.根据物品的相似度和用户的历史行为给用户生成推荐列表。
                        • 改进：
                          • 活跃用户对物品相似度的贡献应该小于不活跃的用户，IUF参数
                          • 归一化：提高准确度、多样性、覆盖率
                          • 哈利波特问题，惩罚热门物品，提高覆盖率和新颖度
                          • 父母喜欢看新闻联播和8点黄金档电视剧，两个不同领域的最热门物品之间具有比较高的相似度怎么办呢？
                        • 目前业界运用最多；推荐基于个人历史行为，更个性化；图书、电子商务、电影。
                    • 隐语义模型 （latent factor model）
                      • 分类、负样本采样，迭代计算用户u对物品i的兴趣
                      • 很难实现实时推荐
                    • 基于图的随机游走算法（random walk on graph）
                      • 用户物品二分图，推荐任务转化为度量图中两个顶点的相关性
                      • PersonalRank算法：多次随机游走,节点被访问到的概率会收敛到一个数。
                        • 缺点：时间复杂度，每个用户都需要迭代一遍
                        • 解决：1.减少迭代次数；2.转化为概率转移矩阵，利用矩阵论直接求值。
                  • 评测：将用户行为数据集按照均匀分布随机分成M 份(本章取M=8)，挑选一份作为测试集，将剩下的M-1份作为训练集。
                    • Recall、Precision、覆盖率、新颖度

                  # 利用用户标签数据

                  • UGC标签：Delicious 给网页打标签、CiteULike 给论文打标签、Last.fm、豆瓣、Hulu、
                  • 问题：标签推荐、基于标签的推荐
                  • 基于标签的推荐
                    • Simple 算法：首先找到用户常用的标签，然后找到具有这些标签的最热门物品推荐给这个用户。
                      • 改进:
                        • 热门标签、热门物品权重过大，借鉴 TF-IDF 改进
                        • 数据稀疏，进行标签扩展（基于标签相似度）
                        • 标签清理（高频停止词、同义词等）
                      • 用图模型解释
                  • 标签推荐
                    • 结合 物品i上最热门的标签 和 用户自己经常使用的标签
                    • 冷启动问题
                      • 内容中抽取关键词
                      • 标签扩展
                    • 图游走
